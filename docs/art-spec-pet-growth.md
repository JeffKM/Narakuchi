# 아트 스펙 — 펫 생애단계 (육성) · 첫 펫 = 시온이

> 설계 단일 출처: **ADR 0005**(펫 육성 = 분기형 진화) · 도트 규격: **ADR 0001**.
> 아트가 코드보다 선행. 첫 슬라이스 = 시온이. 나머지 펫은 이 스펙을 미러.

## 1. 무엇을 그리나 (분기 매트릭스)

같은 **아기** → (간식/놀기 밸런스) → **성체 3분기**. 중간 **유년**은 단일(분기 미노출).

| 단계 | 분기 | 반응 4컷 | 비고 |
|---|---|---|---|
| 아기 (baby) | 1 (공통) | idle/snack/play/pet | 작고 동글, "꼬물이" |
| 유년 (child) | 1 (공통) | idle/snack/play/pet | 분기 전 중간형 |
| 성체 마른 (thin) | 분기 | idle/snack/play/pet | 기존보다 **더 마르게** (날렵·슬림) — 신규 |
| 성체 보통 (normal=**캐논**) | 분기 | idle/snack/play/pet | **기존 `sioni_*` 재사용** (이미 통통-귀여운 캐논) |
| 성체 뚱뚱 (fat) | 분기 | idle/snack/play/pet | 기존보다 **더 통통하게** — 신규 |

**신규 아트 = 16컷** (아기4 + 유년4 + 마른4 + 통통4). 보통(캐논) 4컷은 *기존 시온이 재사용* → 0.
> 캐논 재배치(2026-06-21 grill): 실제 시온이가 통통한 건 맞으나 **기존 배포 그림이 이미 충분히 통통-귀여워 그대로 대표(보통=캐논)로 승격**. 양 끝으로 *더 마름*·*더 통통*을 신규로 그린다. (이전엔 뚱뚱=캐논·기존 재사용이었음 → 뒤집힘. ADR 0005 동기화.)
> 모든 분기는 "같은 흰 얼룩 고양이"로 식별돼야 함. 차이는 **체형/체급(말랐나·통통하나)**뿐. 얼굴·무늬·색은 공유.

## 2. 규격 (전 단계 공통 — ADR 0001 사수)

- **캔버스 `96×96`** 고정. 단계가 작아도 캔버스는 96×96 — *아기는 캔버스 안에서 작게* 차지(여백 큼), 자랄수록 캔버스를 더 채운다.
  - **크기 위계는 픽셀에 굽는다**(런타임 scale 아님 — 프로젝트 관례: 물리 크기는 PNG에, 코드는 native scale 1.0). `tools/dotify.py --fill-scale`(부분 충전, 발은 바닥 고정)로 단계별 콘텐츠 높이를 정한다. **목표 비율(콘텐츠 높이 ÷ 96)**: **아기 50% · 유년 68% · 성체(마름·통통) 82%**(= 기존 캐논 `sioni_idle` 과 같은 "지금 크기"). `--fill-scale` = 목표 ÷ 0.96. 도트 스튜디오 A16 슬롯이 이 값을 자동 세팅. D1 진화 엔진은 prefix 스왑만으로 즉시 크기 전환(하드컷 reveal = 성장 임팩트).
- **피벗 = 발밑(하단 중앙)**. 성장이 *바닥에서 솟아오르듯* 보이게(스케일·홉이 발에 붙음). 좌상단 정렬은 코드가 `(-48, -96)`로 끌어올림.
  - **발밑 여백 = 캐논과 정합**: 시온이는 디오라마 받침에 앉아 발이 캔버스 바닥이 아니라 **row 86(발여백 10px)**에 온다(캐논 `sioni_idle`·`cafe.gd SIONI_PAD_BOTTOM=11`·그림자 위치). 단계 스프라이트도 **`--foot-margin 10`**(도트 스튜디오 자동)으로 같은 발 높이에 굽지 않으면 캐논보다 아래로 내려가 그림자와 어긋난다.
- **NEAREST**, 1텍셀=1픽셀 정수. scale 1.0 + 줌 ×2 모두 정수배.
- **마스터 팔레트 인덱싱**(~49색) — `tools/dotify.py` 파이프라인(AI 도트화 → 축소 → 팔레트 인덱싱 → 크로마키 분리). 신규 색 금지(벗어나면 `audit_sweep` 위반).
- **누끼**(투명 배경) — 라이브 스프라이트는 배경 없음(디오라마 위에 합성). 체키만 베이크컷.
- 포즈 규칙(Okja/Sioni 계승): idle=둥실 흔들 기준, snack/play/pet=반응 정착. **하드컷 스왑** 전제(크로스페이드 금지 — 자세 달라 고스팅).

## 3. 네이밍

`{pet}_{stage}_{react}.png` — 레지스트리 주도 일반화(전 펫 동일 규칙).

```
sioni_baby_idle.png   sioni_baby_snack.png   sioni_baby_play.png   sioni_baby_pet.png
sioni_child_idle.png  sioni_child_snack.png  sioni_child_play.png  sioni_child_pet.png
sioni_thin_idle.png   sioni_thin_snack.png   sioni_thin_play.png   sioni_thin_pet.png   # 더 마름 (신규)
sioni_fat_idle.png    sioni_fat_snack.png    sioni_fat_play.png    sioni_fat_pet.png    # 더 통통 (신규)
# 보통(normal=캐논) = 기존 sioni_idle/snack/play/pet 재사용 (코드에서 stage=normal → 접두어 없는 기존 파일로 폴백/alias)
```
> 현 코드 `sprite_prefix`(`{prefix}_{react}`)를 `{pet}_{stage}` 형태로 확장. normal은 레거시 `sioni_*` 매핑으로 무손상 호환.

## 4. 진화 연출 (성장 단계 도달 = 체키 아님, 희소 임팩트)

**프레임 시트 금지(ADR 0001).** 코드 트윈 + 기존 이펙트 재사용으로:
- 시퀀스(약 1.5s): 펫 스쿼시 → **화이트 실루엣 플래시**(스프라이트 흰색 틴트 + 스케일 팝, 교체를 가림 = 포켓몬식) → 그 아래 텍스처를 다음 단계로 **하드 스왑** → 플래시 아웃 + **반짝임/광선 버스트**(`burst_rays.gd`·`sparkle` 재사용) → 정착 스쿼시.
- 성체 진화 순간 = *밸런스로 결정된 분기*가 공개되는 순간(마른/보통/뚱뚱 reveal).
- 티커 한 줄("시온이가… 자랐어!") + 폴짝(hop). 추가 아트 **0**(스테이지 스프라이트 외).

## 5. 산출/검수

- 위치: `assets/sprites/` , 임포트 후 `tools/verify_cheki_art.gd`(펫 순회)·`tools/audit_sweep.py`(팔레트·규격 53+) 통과.
- DoD: 16 신규 컷 임포트·알파·팔레트·96×96·발밑 정렬 검수 → 헤드리스 로드 무에러 → 디오라마 ×1/×2 렌더 또렷.
- 특수형 분기(천사·나비 등)는 **이 스펙 범위 밖**(전 펫 아기→성체 완성 후, 나비 승급식 덧입히기 = ADR 0005 T45).

## 6. 다른 펫으로 미러 (이후 슬라이스)

규종이·코코·선아·수아도 동일 매트릭스(아기4+유년4+마른4+보통4, 캐논=기존 성체 재사용). 펫별 캐논 체형이 다름(코코=마른 게 캐논일 수 있음 → 그 경우 재사용 분기만 바뀜). 펫당 신규 ~16컷.
