# Gemini 도트 변환 프롬프트 가이드 (허브)

> 워크플로우 B: **AI는 형태·색감만, 규격은 `tools/dotify.py`가 강제.** (→ [tools/README.md](../tools/README.md))
> ADR 0001 파이프라인 = **실물 사진 → AI 도트화**(img2img). 상상 생성이 아니라 사진 변환이 기본 — 인물 고증·일관성을 위해.
>
> 가이드가 커져 **캐릭터별 + 공통**으로 분리했다. 작업할 대상의 파일만 보면 그 캐릭터 한 명을 끝까지 뽑을 수 있다(각 파일이 프롬프트 + 후처리 명령까지 자체 포함).

## 공통 규칙 (요약 — 상세는 [공통 파일](./gemini-prompts-common.md))

1. **사진 변환 우선** — 실물 사진 첨부 img2img. 텍스트만이면 얼굴이 흔들린다.
2. **단색 배경(크로마키)으로 받는다** — 기본 크로마 그린 `#00ff00`, 초록 의상(크리스마스)이면 마젠타 `#ff00ff`. `dotify --chroma`로 분리.
3. **정확한 px·색 수는 요구하지 않는다** — 구도·비율·무드만, 규격은 후처리(`dotify`).
4. **표정 6종 = 다리·구도 고정, 얼굴 + 팔 자세만 변경**.

## 반복 루프

1. **(나)** 프롬프트 제공 → 2. **(너)** Gemini에 사진+프롬프트 → 결과 PNG → 3. **(나)** `dotify` 규격화 + 검수 → 4. 미달이면 수정안 → 2 반복 → 5. 통과 → pixilart 수동 정리 → 확정.
