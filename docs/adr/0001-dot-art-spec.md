# 도트 아트 규격: 세로 캔버스(내부 480px) + 마스터 팔레트 + 사진→도트 파이프라인

> 갱신 2026-06-04: 게임기 셸을 도트풍 레퍼런스로 교체하며 캔버스/내부화면 치수가 바뀜. 본문은 현행 규격, 변경 경위·근거는 맨 아래 **§갱신 이력** 참조.

## 결정

게임은 **세로 캔버스, 정수배 스케일, Nearest 필터**로 렌더링한다. 콘텐츠 도트 밀도 기준은 **세로 480 px 1배**다(가로는 화면 비율에 따름). 전체 표시 캔버스는 **게임기 셸 포함 633×875**, 내부 교감화면은 **셸 LCD 구멍 333×480**이다(초기엔 270×480 9:16을 베이스로 잡았다 — §갱신 이력). 모든 에셋은 나라카 브랜드 앵커(먹빛 블랙·버건디·블러드레드·앤틱 골드·캔들 옐로 등) 기반 **약 32색 마스터 팔레트로 인덱싱**해 통일한다. 아트는 직접 픽셀을 찍지 않고 **실물 이벤트 사진 → Midjourney/Gemini 도트화 → 목표 해상도로 nearest-neighbor 축소 → 마스터 팔레트 인덱싱 → 수동 정리** 파이프라인으로 생산한다. 한글 도트 폰트는 **갈무리11/9**.

주요 스프라이트 규격: **옥자(라이브 메인) 전신 약 128×288 — 프레임 애니/리깅이 아니라 "표정 스왑 스탠딩"(얼굴 + 팔 자세가 다른 6장을 통째로 교체, 다리·하체·프레이밍은 고정) + 코드 트윈(평소 둥실 흔들·페이드, 표정 전환 시 하드컷 + 스쿼시 정착)으로 구동**, 시온이(교감 가능한 펫) 48×48, 체키 카드 120×180(내부 아트 영역 104×140 고정), UI 아이콘 24×24, 재화 아이콘 16×16. 배경은 **나라카 지옥 배경(세계관 — 지옥 풍경)**, 내부 교감화면(333×480)에 맞춘다. **게임기 셸은 도트풍 레퍼런스(`assets/sprites/_src/damagochi_frame.png`)를 채택**해 `tools/prep_shell.py`로 흰배경·LCD 안쪽 투명화 + 리샘플한 **633×875 프레임**(다마고치형 달걀 바디 + 목걸이 고리 + 불꽃 + N 브랜드 + 동글 3버튼 SELECT(이동)·OK(확인)·CANCEL(취소)). 내부 교감화면은 **셸 LCD 구멍 333×480에 꽉 맞추고**(크롭/여백 없음) 셸 프레임을 콘텐츠 위에 얹어 베젤로 가장자리를 덮는다. 셸 바깥은 **완전 투명**(뷰포트 `transparent_bg`). LCD 위치·버튼 좌표는 `prep_shell.py` 계측값을 `scripts/systems/shell.gd` 상수와 동기화한다.

> 라이브 캐릭터를 프레임 시트로 그리거나 컷아웃 리깅하면 작업·난이도가 커진다. 미연시가 쓰는 스탠딩+표정차분 방식이 가장 쉽고, 본편 미연시와 에셋도 호환된다.
>
> **표정 6장 = 얼굴 + 팔 자세 변경 (다리·하체·프레이밍은 고정).** 팔 자세도 그림에 박아 두는 **아트 레벨**이다 — 팔을 별도 레이어로 떼어 코드로 굴리는 컷아웃 리깅은 **금지**(앵커 어긋남·이음새·작업량 폭증). 팔 자세가 표정마다 다르므로 알파 크로스페이드는 반투명 도트가 겹쳐 팔레트 밖 중간색이 뜨는 **고스팅**이 난다 → 전환은 **하드컷 + 짧은 스쿼시 정착 트윈(0.1~0.15s)**으로 통일(페어별 튜닝 불필요, 컷이 오히려 "반응" 생동감). 기쁨 "팔 벌려 폴짝" 같은 **큰 전신 포즈는 별도로 그리지 않고**, `okja_smile`(두 손을 가슴께로 모아 기뻐하는 자세)을 체키 획득·나비 승급 등 리워드 순간에 **코드 hop 트윈**으로 재사용해 표현한다. **우는 연출은 「벌 없는 설계」상 금지** — 슬픔의 바닥은 **팔 축 처진 시무룩**까지다(→ PRD 리스크).

## 이유 / 트레이드오프

- **세로 480 밀도**: 8비트 순수 레트로(예: 180×320)는 옥자님 얼굴·의상 디테일이 뭉개져 "감정 임팩트" 목표와 충돌. 반대로 고해상도는 도트 감성·작업 통일성을 잃음. 세로 480은 얼굴 인식과 레트로 감성의 균형점이다. (전체 캔버스는 셸 채택으로 633×875로 바뀌었으나 도트 밀도·integer scale·Nearest는 그대로라 픽셀은 또렷이 유지 — §갱신 이력.)
- **마스터 팔레트 강제**: AI 도트화는 색이 제각각 튀어 한 게임처럼 안 보임. 인덱싱으로 통일감을 강제한다.
- **사진→도트 파이프라인**: 무예산·1주·일러스트레이터 미확보 상황에서, 카페가 이미 보유한 1년치 이벤트 사진을 활용하는 유일하게 현실적인 길. 고증·속도·일관성을 동시에 확보.

## 되돌리기 비용 (이 결정을 ADR로 남기는 이유)

해상도와 팔레트는 **한 번 정하면 전 에셋이 종속**된다. 나중에 바꾸면 모든 도트를 재작업해야 하므로, 초기에 못 박고 그 이유를 남긴다.

## 갱신 이력

### 2026-06-04 — 게임기 셸을 도트풍 레퍼런스로 교체, 캔버스/내부화면 치수 변경

**무엇이 바뀌었나**

- 게임기 셸: 절차생성 460×630(좌우 베젤 95 / 상단 브랜드 바 30 / 하단 버튼 덱 120) → **도트풍 레퍼런스 채택 → `tools/prep_shell.py` 가공 633×875 프레임**.
- 전체 표시 캔버스: 270×480 → **633×875**.
- 내부 교감화면: 270×480(9:16) → **셸 LCD 구멍 333×480**(가로만 확장, 세로 480 유지).
- 셸 바깥 여백: 다크 버건디 배경 → **완전 투명**.

**왜**

- 무예산·1주 제약에서 절차생성 셸은 도트 질감이 없어 "감정 임팩트 + 공유성" 목표에 미달. 실제 도트풍 레퍼런스가 훨씬 매력적이라 채택.
- 레퍼런스 LCD 비율(≈3:4)이 콘텐츠(9:16)와 달라, **내부 화면을 셸 LCD 구멍에 꽉 맞춤**(크롭/여백 없음)을 선택(사용자 결정).

**유지된 것 (규격의 근간 — 불변, 재작업 불필요)**

- 콘텐츠 도트 밀도(세로 480 1배) · Nearest 필터 · integer scale.
- 마스터 팔레트 32색 인덱싱 · 실물 사진 → AI 도트화 파이프라인.
- 옥자 128×288 표정 스왑 스탠딩(얼굴 + 팔 자세 6장, 다리·구도 고정) · 시온이 48×48 · 체키 카드 120×180(아트 104×140) · 갈무리 폰트.

**영향 / 재현**

- 캐릭터·카드 에셋 도트 밀도는 영향 없음(세로 480 기준 동일).
- 셸 재생성: `python tools/prep_shell.py` (원본 `assets/sprites/_src/damagochi_frame.png` → `assets/sprites/shell_frame.png`).
