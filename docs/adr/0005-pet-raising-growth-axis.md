# 펫 육성을 성장축으로 — 분기형 진화 + 되돌릴 수 있는 돌봄 압박

> 합의 2026-06-21(grill-with-docs). 데모 검증 이후, 나라쿠치를 *카페 방문자가 매일 여는 앱*으로 키우는 방향. 미연시는 별개 트랙.

## 결정

성장·리텐션 엔진을 **역할 분담**으로 설계한다.

- **메인 = 호감도(관계·수집)**, **펫 = 육성(키우는 재미)**. 캐릭터 본체는 정해진 IP라 "키우는" 대상이 될 수 없고, 관계 *심화*는 미연시 영역이라, *키우는 성장*은 펫이 가져간다.
- **펫 육성 = 분기형 진화**. 같은 *아기*에서 시작 → **돌보는 방식**(간식 vs 놀기 *밸런스*)에 따라 성체 체형이 분기(마른/보통/뚱뚱). **대표·캐논 = 보통(기존 배포 시온이 그림)** — 실제 시온이가 통통한 건 맞으나 기존 그림이 이미 통통-귀여워 그대로 대표로 승격, 양 끝의 *더 마름*·*더 통통*만 신규로 그린다(2026-06-21 grill 재배치 — 이전 "뚱뚱=대표·기존 재사용"을 뒤집음, art-spec 동기화). **체형은 성체 도달 시 확정·영구**(내가 키운 고유한 결과 = 공유성·애착). 이후 *특수형 분기*(천사·나비 등)는 최후 확장.
- **돌봄 압박 = "되돌릴 수 있는 부드러운 압박"**(네코 아츠메·Finch 류, 다마고찌의 *벌* 없이 *손맛*만).
  - 불변 규칙: **압박은 "하루 한 번 들르면 완전히 복구되는 표면"(🟢 = 기분·배고픔·카페 정돈)에만 작용하고, *획득·적립*한 것(🔴 = 보유 체키·관계 단계·누적 호감도·자란 성장단계)은 절대 건드리지 않는다.** 죽음·이탈·병 없음.
  - 한 줄 경계: *오늘 안 와서 잃는 건 '오늘의 좋은 것'(체키 한 장·기분 좋은 메인)뿐, 어제까지 쌓은 건 안전하다.*
  - 컨디션(🟢)은 성장 *속도*만 조절(시무룩하면 굼뜸, 회복하면 쑥쑥) — *역행 0*. 체형 *분기*는 안 건드림(직교: 컨디션=속도, 밸런스=모습).
- **두 트랙은 따로 돈다**(한 스태미나 풀을 쪼개는 게 아님).
  - 메인: **하루 한 명** 선택해 호감도 채움("오늘은 옥자? 미호?"). 게이지 차면 그 메인 체키 → 내일까지 잠금.
  - 펫: *선택이 아닌 의무* — 활성 펫 하나를 돌봄. 진척 게이지 차면 펫 체키.
- **체키 소스 분리**: 메인 게이지→메인 체키 / 펫 게이지→펫 체키(둘 다 호감도·진척 *일정량마다* 촬영). **펫 성장 단계 도달 = 체키가 아니라 *진화 연출 장면***(단계가 몇 개뿐이라 체키론 빈약 → 희소한 *기억의 순간*으로). 데모/디버그는 무한 연타 허용(테스트), 실제는 게이트.
- **한 번에 한 마리만 육성**. 성체 완성 시 펫은 **졸업** → 카페에 *상주*(더는 매일 의무 아님, 가끔 반응) → 다음 펫 해금해 육성. 매일 의무는 늘 ~1마리 = 가벼움 유지. *졸업한 펫이 카페에 늘어남* = 공간이 채워지는 부차 성장.
- **♡ 경제(T39)를 데일리 당김의 심장으로**: 컬렉션은 *언제든 100% 달성 가능*(🔴 보호) + 매일 들르면 ♡·streak를 적립, 빠지면 *카드가 아니라 그 보너스*를 놓침(하이브리드 FOMO). ♡로 다음 펫/메인을 *확정 해금*(랜덤 아님 = 무과금).

## 이유 / 트레이드오프

- **"다마고찌답게"(매일 챙기는 손맛) ↔ `벌 없는 설계`(브랜드 안전)의 정면 충돌**을 *되돌릴 수 있는 압박*으로 통과. 잃는 게 없으니 안전하면서, "나 없으면 안 돼" 훅은 산다.
- **펫을 성장축으로** 둬서 ① IP 훼손(메인을 아기로 못 만듦)과 ② 미연시 중복(관계 심화)을 동시에 회피. 펫은 동물이라 체형 변주가 캐논 안전선 안.
- **분기형 + 성체 확정**: "내 시온이는 너랑 달라"의 공유성·애착. 한 번에 한 마리·졸업제로 *가벼움*과 *완성의 성취*를 동시에.
- **성장=진화 연출(체키 아님)**: 희소 단계를 임팩트로, 잦은 체키는 게이지로 — 보상 밀도를 소스별로 분리.
- **두 트랙 분리**: 메인의 *선택*("오늘 누구")과 펫의 *책임*("내 새끼")이 다른 결의 리텐션을 만들어 매일 *둘 다* 열 이유가 생김.

## 폐기된 대안

- **(가) 무압박**(현행: 방치해도 아무 일 없음) → "와야 할 이유"가 약해 리텐션 부족.
- **(다) 진짜 다마고찌**(병·이탈·죽음) → `벌 없는 설계`·브랜드 안전 위반.
- **메인 캐릭터 육성** → 정해진 IP를 아기로 만들 수 없음.
- **공간/컬렉션만 성장축** → "키운다"는 다마고찌 정서가 약함(졸업 펫 상주로 부차 흡수).
- **공유 스태미나(메인↔펫 한 예산 배분)** → 트랙을 따로 두는 게 두 결의 리텐션에 유리.
- **펫 전부 매일 의무 / 동시 다수 육성** → 마리수 늘수록 숙제화, 한 생명 애착 희석.

## 데모 진화 슬라이스 (선반영 — 2026-06-21 grill)

배포된 데모를 *리텐션 앱*으로 검증하기 **전에**, 진화의 *감정 임팩트 + 공유성*만 먼저 데모에 얹는다. Phase 8의 풀 설계에서 다음을 **데모 모드로 단순화**한다:

- **페이싱 = 횟수 기반·한 세션 완주 허용(A)**. 돌봄 압박(🟢 컨디션=속도, T40)·하루 진행 제한·한 마리씩·졸업은 **데모 이후 리텐션 단계로 미룸**. → 데모에선 *속도축을 끄고 밸런스축만* 살림(나중 T40 얹으면 속도축 복원).
- **분기 3종 데모 포함**. 성장 미터 = **간식·놀기·쓰담 3버튼**으로 채움(체키 제외). 체형 = 누적 *간식 vs 놀기* 치우침으로 성체 도달 순간 확정, 비등=보통, 쓰담 중립. 임계·단계수 = `data/balance`.
- **체키 버튼 유지**(시온이 체키 그대로). 성장 단계 도달 = **진화 연출**(체키 아님, `burst_rays`·`sparkle` 재사용 = 추가 아트 0).
- **HUD: 펫 게이지를 "성장 미터"로 용도 변경**(메인 옥자 호감도 게이지 무손상). 좁은 세로 화면에 바 1개 유지.
- **저장 = 기존 `SaveManager` 그대로**(리셋 로직 없음). 데모는 웹 링크라 방문자별 폰 localStorage = 각자 아기부터.

## 향후 확장 / 미결(나중 튜닝)

- ♡ 적립·해금 비용, 성체까지 일수(스태미나 예산당 성장량) = `data/balance` 수치.
- 특수형 분기는 *전 펫 아기→성체 완성 후* 마지막에. *나비 승급*처럼 캐논에 덧입히는 치장으로 처리(IP 충돌 회피).
- 방치→캐치업 먹이기가 통통 분기로 미는 창발 = 의도된 맛/중립 중 밸런싱 때 결정.
- 졸업 펫의 카페 상주 연출, 메인↔펫 교차 호감도(옥자 펫 돌보면 옥자 기뻐함) = 여지로 남김.
