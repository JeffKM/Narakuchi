# 카드형 체키: 런타임 합성 양면 카드(앞=표지, 뒤=사진) + 표지 등급 엠블럼

> 신규 2026-06-05. ADR [0002](./0002-cheki-collection-model.md)(프레임-only 등급)를 **렌더링·연출 차원으로 확장**한다. 수집 모델(캐릭터×이벤트 세트축 + 등급=프레임)은 0002 그대로 유효하다.

## 결정

체키를 **눌러서 뒤집는 양면 카드**로 정의하고, **구운 1장 PNG가 아니라 런타임 레이어 합성**으로 렌더한다.

- **2면 구성** (실물 4면 접지를 데모용 2면으로 압축):
  - **앞면(표지, 기본 노출)** = 파치먼트 배경 + **등급 엠블럼**(날개/나비) + **나라카 붓글씨 워드마크** + **닉네임·날짜**(갈무리 폰트 런타임 렌더). 손글씨 헌사 느낌.
  - **뒷면(뒤집어야)** = 의상 **사진**(캐릭터 아트) + **뒷면 사진 프레임**. **큰 폴라로이드 레이아웃**: 균일 얇은 테두리 `~6px` + 큰 사진 창 `~108×162`(카드의 ~90%) + 붉은 드립(상단). **별도 캡션 스트립 없음** — 데이 라벨("○○ 데이" = `Events.cheki_day_label`, 표시명에서 파생)은 **런타임에 사진 창 윗쪽에 오버레이**(실물 체키가 사진 위에 손글씨 적던 자리, 갈무리). 표지의 닉네임·날짜와 별개. 로고·QR 없음. 의상 스탠딩(`128×288`)은 **상반신 앵커로 크롭**해 사진 창을 채운다(스냅샷 느낌). 단 의상 아트는 **누끼(투명 배경)** 라 그대로 두면 사진 창 뒤가 빈다 → **이벤트(의상)에 종속된 배경 레이어**(`bg_cheki_*`, `120×180` 불투명)를 의상 누끼 **뒤**에 깔아 스냅샷 현장을 만든다. 즉 **사진 면 = `[배경 레이어] + [의상 누끼] + [사진 프레임 테두리]` 3겹 합성**. 등급이 프레임을 표준↔테마로 스왑해도 **배경은 의상에 묶여 항상 유지**(나비 승급은 테두리만 바뀜).
- **등급(일반/나비)은 표지 엠블럼의 변태로 드러낸다**(실물 나라카 로고 기반, 레퍼 `assets/sprites/_src/naraka_logo_ref.png`): **일반 = 비대칭 쌍날개**(왼쪽 깃털 완전날개 + 오른쪽 그을린·구멍난 탄날개) + 붉은 잉크 스플래터, **나비 = 그 쌍날개가 온전한 대칭 지옥풍 나비로 변태(탄날개가 아물어 부화) + 앤틱 골드**. "중복 → 나비 승급"이 표지에서 문자 그대로(탄날개→나비 부화) 일어난다. 등급은 **2종 바이너리 유지**(쌍날개는 일반 엠블럼 한 장 안의 좌우 비대칭이지 별도 등급이 아니다 — ADR 0002 그대로). 뒷면 사진 프레임도 ADR 0002대로 표준→테마로 교체되어 **나비체키 = 나비 표지 + 테마 프레임**(더블 보상).
- **뒤집기 = 가짜 3D 가로 플립** (`scale_x: 1→0→1` + 중간 텍스처 스왑, Tween만, 셰이더 없음 → ADR 0001 도트·Nearest 유지).
- **노출 맥락**:
  - **컬렉션 그리드** = **사진 면**을 보여 의상으로 구분(표지가 전부 공용이라 그리드가 똑같은 로고 벽이 되는 걸 방지).
  - **뒤집기**는 **획득 팝업 + 풀카드 확대**에서만.
- **풀카드 확대 입력**(다마고치 3버튼 + 터치 하이브리드 → ADR 0001): `OK`=뒤집기 · `SELECT`=다음/이전 카드 · `CANCEL`=뒤로 · **[공유]=화면 내 터치 버튼**(OS 공유시트 성격). 카드 탭=뒤집기.
- **획득 리빌** = 표지 등장 → 닉네임·날짜가 적혀 들어감(박슥자가 이름을 적는 미세 비트) → `OK`/탭으로 뒤집어 의상 공개(미입력 ~2초 후 자동 플립으로 루프 안 막힘). 나비체키면 지옥풍 나비 파티클.
- **공유** = **현재 보이는 면**을 Viewport 캡처 → Image, @나라카 워터마크 합성. (의상 면도, 내 이름 적힌 표지도 둘 다 공유 가능.)
- **보류(데모 컷)**: QR(표지에 빈 레이어 자리만 남김) · 손편지 · 4면 접지 연출.

## 이유 / 트레이드오프

- **양면 = 손맛 + 소유감 + 공유성**: "표지를 열어 내 의상이 현상되는" 언박싱 리빌이 승리 조건(감정 임팩트 + 공유성)에 직결. 닉네임 적힌 표지는 그 자체로 SNS 자랑거리.
- **런타임 합성(구운 PNG ❌)**: 닉네임·날짜가 플레이어별 동적이라 한 장으로 못 굽는다. 레이어 합성이면 텍스트는 갈무리 런타임, 등급은 엠블럼/프레임 스왑만, **카드별 구운 파일 관리가 0**. 공유는 Viewport 캡처로 동일 파이프라인 재사용.
- **표지 등급 엠블럼(날개→나비)**: ADR 0002가 경계한 "VFX-only 등급 차별"이 아니라 **의미(변태=승급 서사) 중심**이라 0002 정신에 부합. 추가 아트는 **공용 4점**(표지 파치먼트 + 날개 + 나비 + 워드마크)뿐 — 이벤트·캐릭터 무관이라 작업량이 배수로 늘지 않는다.
- **그리드=사진 / 표지=공용**: 표지를 전 카드 공용으로 두어 싸게 가되, 그리드는 사진 면을 노출해 "뭘 모았나"가 한눈에. 뒤집기의 손맛은 팝업·확대에만 집약.
- **셰이더 없는 가짜 3D 플립**: 진짜 3D 회전은 셰이더가 필요하나 ADR 0001이 셰이더를 금지. `scale_x` 스쿼시 + 텍스처 스왑이면 도트·Nearest를 지키며 충분한 카드 물성.

## 폐기된 대안

- **실물 4면 접지 완전 재현** → 접고 펴는 연출·4면 레이아웃·아트가 1주 데모에 과함. 핵심 감정(언박싱·개인화)은 2면으로도 산다.
- **획득 시 양면 굽기(캐시)** → 단순하나 세이브 용량↑ + 닉네임 변경 대응이 번거롭고, 런타임 합성 대비 이점이 작음.
- **공유=항상 사진 면 고정** → 개인화 표지 자랑 경로를 닫음. "현재 보이는 면" 캡처가 더 유연.
- **QR·손편지 데모 포함** → QR은 마케팅 다리로 가치 있으나 보류 레이어로 남겨 나중에 켜기 쉽게, 손편지는 플레이어별 손글씨 아트가 불가(닉네임 동적)라 갈무리 텍스트 헌사로 대체.

## 영향 (구현 메모)

- **데이터 모델(T12)**: 체키 = `{character, event, grade, nickname_snapshot, acquired_at}`. 그림은 들고 있지 않고 합성 시 조립. 사진 면 배경은 `event`에서 파생(`bg_cheki_{event}`), 프레임은 `grade`에서 파생(`frame_standard`/`frame_{event}`).
- **아트(→ [asset-checklist.md](../asset-checklist.md) A2)**: 신규 공용 4점 `frame_cover_bg` · `emblem_wing` · `emblem_butterfly` · `wordmark_naraka`. 뒷면 `frame_standard`는 로고·QR 제거(테두리+드립만)로 정정. **사진 배경 `bg_cheki_*`는 의상(이벤트)별** — 의상 누끼를 받치는 불투명 레이어라 의상 수만큼 필요(공용 아님). 프롬프트 [gemini-prompts.md](../gemini-prompts.md) "체키 카드(양면)" + "체키 사진 배경".
- **연출(T18 획득 / T19 공유)**: 가짜 3D 플립 + 리빌 시퀀스 + Viewport 캡처 내보내기.
