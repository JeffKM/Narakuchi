# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

> ⚠️ **엔진 전환됨**: 이 프로젝트는 더 이상 Next.js/Phaser 고양이 게임이 아니다. **Godot 4** 기반 **나라카찌**로 전면 재시작되었다. 전역 CLAUDE.md의 Next.js/React 스택 규칙은 이 저장소에 **적용되지 않는다.**

## 프로젝트 개요

**나라카찌 (Narakatchi)** — 나라카 컨셉(메이드)카페 팬게임. 도트 그래픽의 가벼운 **데일리 교감 + 체키(포토카드) 수집** 게임. 플레이어는 매일 나라카에 들러 사장 **옥자**(지옥의 마녀)와 교감하며 친해지고, 이벤트 데이 의상의 **체키**를 모은다. 약 1주 작업의 가능성 데모이며, 카페 마케팅과 장차 미연시로의 다리를 목표로 한다.

- **승리 조건**: "감정 임팩트 + 공유성" (깊은 시스템 < 감정 + 공유)
- **플랫폼**: 웹(Godot HTML5 export) · 모바일 세로 · PWA
- **저장**: 로컬(JSON) 전용, 서버/계정 없음

## 명령어

```bash
# 에디터로 프로젝트 열기
godot project.godot

# 헤드리스 실행 / CI
godot --headless --path .

# 웹 빌드 (export_presets.cfg에 "Web" 프리셋 구성 후)
godot --headless --export-release "Web" export/index.html
```

> npm/Node 기반 명령은 없다. Godot 에디터 + GDScript로 작업한다.

## 기술 스택

- **Godot 4** (2D), **GDScript**
- **HTML5 export** + **PWA**(manifest + service worker, 세로 고정)
- **갈무리11/9** — 한글 도트 비트맵 폰트 (라이선스 확인)
- 저장: `localStorage`(JSON 세이브). 셰이더·외부 백엔드 없음.

## 아키텍처

### 게임 해상도 / 셸 (→ ADR 0001)
- 베이스 캔버스 **270×480**(9:16 세로), 정수배 스케일, **Nearest 필터**.
- **게임기 셸**: 화면을 **다마고치형 달걀 바디**(목걸이 고리)로 감싸고, 폰 세로 여백을 **버튼 덱**으로 채움. **다마고치식 동글 3버튼**(SELECT 이동·OK 확인·CANCEL 취소)은 **화면 터치와 함께 실제 기능에 연결**.

### 권장 폴더 구조
```
project.godot
scenes/        # 셸, 온보딩, 교감(메인), 컬렉션북, 공유, 체키획득 팝업
scripts/
  systems/     # 호감도·스태미나·기분·체키·저장·출석
data/          # 이벤트/체키/대사/선물/밸런스 상수 (Resource 또는 JSON)
assets/
  sprites/     # 옥자 표정 스탠딩, 시온이, 체키 카드, 테마 프레임, 게임기 셸
  fonts/       # 갈무리
export/        # HTML5 빌드 산출
```

### 라이브 옥자 = 표정 스왑 스탠딩 (→ ADR 0001)
프레임 시트·컷아웃 리깅 **금지**. 표정/포즈별 5~6장(기본/웃음/부끄/시무룩/제조/말하기)을 통째로 교체 + 코드 트윈(둥실 흔들·페이드)으로 생동감.

### 핵심 흐름
```
온보딩(닉네임 입력 → 첫 지뢰계 체키 증정)
  → 교감(메인): 옥자 4버튼[체키주문·음료주문·대화·선물] + 옥자 터치 / 시온이 탭 → 시온이 4버튼 모드
  → 호감도 게이지 가득 → "오늘의 체키" 자동 획득(중복 → 나비 승급)
  → 컬렉션북(캐릭터×이벤트 칸) → 공유 이미지(@나라카 + QR)
```

## 핵심 규칙

- **수치 상수는 한 곳(`data/` 밸런스 상수)에서 관리** — 하드코딩 금지. 값은 PRD §4.5 참조.
- **도트 규격 사수**: 270×480 / 마스터 팔레트(~32색) 인덱싱 / 체키 카드 120×180(양면, 뒷면 사진 창 ~108×162 + 윗쪽 데이 라벨 → ADR 0001·0003).
- **짧은 대사 원칙**: 표정 9할, 대사 한 줄. 옥자 보이스는 **한 줄 티커** + 선택지는 **`대화` 팝업**에만. 대사는 `docs/script-okja.md` 참조.
- **벌 없는 설계**: 캐릭터는 아프거나 떠나지 않음(시무룩까지만). 마케팅 게임 브랜드 안전.
- **무과금**: 게임 내부 현금 가챠 없음. 현금은 "현실 방문/굿즈 → 한정 체키 해금"의 오프라인 다리로만(데모엔 컨셉).
- **한국어**: 주석·커밋·문서 모두 한국어. 변수/함수명만 영어.
- **들여쓰기 2칸**(GDScript는 탭 기본이나 프로젝트 일관성 위해 2칸 스페이스 권장), 함수/변수 snake_case(GDScript 관례), 클래스/노드 PascalCase.

## 도메인 용어 (CONTEXT.md 참조)

- **옥자**: 사장, 지옥의 마녀. 메인 교감·수집 캐릭터(라이브, 기본=마녀룩). 시크·츤데레·장난기.
- **시온이**: 옥자의 흰 얼룩 고양이. 곁의 교감 가능한 펫 + 수집 캐릭터.
- **체키**: 캐릭터×이벤트 데이로 모으는 포토카드. **일반체키**(표준 프레임) / **나비체키**(이벤트 테마 프레임). 중복은 나비 승급 재료.
- **관계 단계**: 손님(존댓말+님) → 단골(반말+닉네임) → 마음 연 사이. 말투 전환이 핵심 보상.
- **데모 이벤트 5종**: 지뢰계(★히어로)·유치원·힙합·집사·크리스마스. 시온이는 지뢰계(쿠로미)·크리스마스(루돌프).
